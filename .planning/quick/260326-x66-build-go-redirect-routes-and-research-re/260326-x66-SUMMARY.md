## Architecture Notes

The redirect flow is now:
1. Client clicks internal `/go/{slug}` link (no UTM params in HTML)
2. GA4 `affiliate_click` event fires on click
3. Server receives GET /go/{slug}, looks up product by id
4. Server appends UTM params to real destination URL
5. Server returns 302 redirect to external product page

This avoids GA4 self-referral issues (no UTM params on internal links) and centralizes UTM management server-side for easy future updates.
