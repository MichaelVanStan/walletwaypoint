## Spacing Scale

Declared values (multiples of 4, 8-point grid):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline padding, badge internal padding |
| sm | 8px | Compact element spacing, list item gaps, button icon gap |
| md | 16px | Default element spacing, card internal padding, form field gaps |
| lg | 24px | Section padding, card-to-card gaps, content area side padding on mobile |
| xl | 32px | Layout gaps between major sections, header/footer internal padding |
| 2xl | 48px | Major section breaks on homepage (hero to features, features to trust signals) |
| 3xl | 64px | Page-level vertical padding (top of content area, bottom before footer) |

Exceptions: Touch targets for mobile navigation hamburger button and footer links must be minimum 44x44px (WCAG 2.5.8). This uses a non-standard 44px size for tap area only; visual size remains at standard scale values.

**Source:** Standard 8-point grid system. No upstream override.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.6 | Paragraph text, trust page content, privacy policy, editorial standards |
| Label | 14px | 400 (regular) | 1.4 | Navigation links, breadcrumb text, badge text, disclaimer text, footer links, meta info |
| Heading | 24px | 600 (semibold) | 1.3 | Page titles (About, Editorial Standards, Privacy Policy), section headings |
| Display | 36px | 600 (semibold) | 1.15 | Homepage hero headline only |

**Font family:** `var(--font-inter), ui-sans-serif, system-ui, sans-serif` applied via `--font-sans` in `@theme inline`.

**Responsive scaling:**
- Display: 36px on desktop (lg+), 28px on tablet (md), 24px on mobile (< md)
- Heading: 24px on desktop, 20px on mobile (< md)
- Body and Label: No responsive changes

**Source:** D-03 (CONTEXT.md) locks Inter. Sizes chosen for financial content readability -- 16px body is the minimum for long-form YMYL content. Two weights: 400 (regular) for body and label text, 600 (semibold) for headings and display. Size differentiation (14px vs 16px, 24px vs 36px) provides sufficient hierarchy without additional weight variants.

## Color

All colors use OKLCH notation as generated by shadcn/ui CLI v4. The values below are the exact CSS custom properties from the theme.

### Light Mode (ship in Phase 1)

| Role | CSS Variable | OKLCH Value | Approx Hex | Usage |
|------|-------------|-------------|------------|-------|
| Dominant (60%) | `--background` | `oklch(0.99 0.002 240)` | #fafbfd | Page background, content surfaces |
| Secondary (30%) | `--secondary` | `oklch(0.93 0.01 240)` | #e8ecf2 | Card backgrounds, footer background, nav hover states, muted sections |
| Primary | `--primary` | `oklch(0.35 0.1 245)` | #1a3a6b | Primary buttons, heading text color, nav brand text, active breadcrumb |
| Primary foreground | `--primary-foreground` | `oklch(0.98 0.005 240)` | #f5f7fa | Text on primary-colored surfaces |
| Accent (10%) | `--accent` | `oklch(0.55 0.12 175)` | #0d9488 | See reserved list below |
| Accent foreground | `--accent-foreground` | `oklch(0.98 0.005 175)` | #f0fdfa | Text on accent-colored surfaces |
| Muted | `--muted` | `oklch(0.95 0.005 240)` | #eff1f5 | Disclaimer banner background, placeholder page backgrounds |
| Muted foreground | `--muted-foreground` | `oklch(0.55 0.02 240)` | #6b7b94 | Secondary text, timestamps, meta info, disabled nav items |
| Foreground | `--foreground` | `oklch(0.15 0.02 240)` | #1a1f2e | Body text, heading text |
| Destructive | `--destructive` | `oklch(0.55 0.2 25)` | #c53030 | Destructive actions only (none in Phase 1) |
| Border | `--border` | `oklch(0.88 0.01 240)` | #d4dae5 | Card borders, separator lines, input borders |
| Ring | `--ring` | `oklch(0.35 0.1 245)` | #1a3a6b | Focus ring on interactive elements |

### 60/30/10 Distribution

- **60% Dominant (`--background`):** Page background, main content area, hero background
- **30% Secondary (`--secondary`, `--muted`, `--border`):** Card surfaces, footer, nav bar background, sidebar regions, separator lines, disclaimer banners
- **10% Accent (`--accent`):** Reserved for the explicit list below

### Accent Reserved For (exhaustive list)

1. "Explore Calculators" CTA button on homepage hero
2. "Coming Soon" badge accent dot/border on placeholder pages
3. Links within body content (editorial standards, about page cross-references)
4. Author card "Our editorial process" link
5. Footer social media icon hover state
6. Breadcrumb current-page indicator underline

Accent is NOT used for: navigation links (use `--foreground`), headings (use `--foreground`), card borders (use `--border`), buttons other than the primary homepage CTA (use `--primary`).

### Dark Mode

Dark mode CSS variables are included in `globals.css` for future-proofing (per RESEARCH.md Open Question 3) but NO toggle or `prefers-color-scheme` detection ships in Phase 1. POLISH-02 is explicitly deferred to v2.

**Source:** D-02 (CONTEXT.md) locks blue-based palette with teal/green accent. Exact OKLCH values from RESEARCH.md Pattern 6. D-01 (CONTEXT.md) establishes clean/professional mood.

## Layout Contract

### Responsive Breakpoints

| Breakpoint | Width | Layout Changes |
|------------|-------|----------------|
| Mobile (default) | < 640px | Single column. Hamburger menu. Stacked footer. Display text at 24px. Content padding 16px horizontal. |
| Tablet (sm) | >= 640px | Single column. Hamburger menu. Footer begins 2-column grid. Content padding 24px horizontal. |
| Tablet-landscape (md) | >= 768px | Desktop nav visible. Footer 2-column grid. Display text at 28px. Content max-width 720px centered. |
| Desktop (lg) | >= 1024px | Full desktop nav with dropdowns. Footer 4-column grid. Display text at 36px. Content max-width 720px centered. |
| Wide (xl) | >= 1280px | Same as lg. Max content width remains 720px. Increased side margins. |

### Header

- **Position:** `sticky top-0 z-50` with `backdrop-blur-sm bg-background/95 border-b border-border`
- **Height:** 64px (desktop), 56px (mobile)
- **Content:** Logo (left) + Navigation links (center/right on desktop, hidden on mobile) + Hamburger icon (mobile only, right)
- **Logo:** Text-based "WalletWaypoint" in `--primary` color, 18px semibold. No image logo in Phase 1.

### Footer

- **Background:** `--secondary`
- **Structure:** 4-column grid (lg+) with columns: Tools, Learn, Company, Legal. Below columns: full-width financial disclaimer in `--muted-foreground` at 12px. Copyright line at bottom.
- **Padding:** 48px vertical, page content horizontal padding

### Content Area

- **Max width:** 720px for text-heavy trust pages (About, Editorial Standards, Privacy Policy). Centered with auto margins.
- **Homepage:** Full width with sections constrained to 1200px max-width.
- **Vertical rhythm:** Headings have 32px margin-top and 16px margin-bottom. Paragraphs have 16px margin-bottom. Sections separated by 48px.

### Disclaimer Banner (D-12)

- **Position:** Fixed bottom of viewport OR top of content area (below header, not overlapping)
- **Recommended:** Top of content area, below header, as a full-width muted bar. This avoids covering content at the bottom and CLS issues.
- **Height:** Auto (approximately 40px with padding)
- **Background:** `--muted` at 50% opacity
- **Dismissible:** No -- always visible on calculator and guide page routes. On trust pages and homepage, the banner is not shown (redundant).

## Interaction Contract

### Navigation

| Interaction | Behavior |
|-------------|----------|
| Desktop nav link hover | Text color transitions from `--muted-foreground` to `--foreground` over 150ms |
| Desktop nav link for disabled/coming-soon | Renders as `--muted-foreground` with `cursor-default`, no hover effect, "(Coming Soon)" label appended |
| Mobile hamburger tap | Opens `Sheet` from left with slide animation, full nav list, close button |
| Breadcrumb link click | Standard navigation. Current page item is non-linked text in `--foreground`. Previous items are links in `--muted-foreground` with underline on hover. |
| Footer link hover | Underline appears, color transitions to `--foreground` over 150ms |

### Homepage

| Interaction | Behavior |
|-------------|----------|
| Primary CTA ("Explore Calculators") click | Navigates to `/calculators` placeholder page |
| Secondary CTA ("Read Our Guides") click | Navigates to `/guides` placeholder page |
| Trust signal cards | Non-interactive. Static display. No hover effects. |
| Placeholder "Coming Soon" cards | Non-interactive. Static display with `--muted` background and `badge` indicator. |

### Placeholder Pages

| Interaction | Behavior |
|-------------|----------|
| "Back to Home" button click | Navigates to `/` |
| Cross-links to trust pages | Standard link navigation |

### Animations (Phase 1 scope)

Minimal animation in Phase 1. Content pages are static -- do not add gratuitous transitions.

| Element | Animation |
|---------|-----------|
| Mobile nav sheet | Default `Sheet` slide-in/slide-out from shadcn/ui (uses Radix Dialog animation) |
| Header sticky | No animation on scroll attach. Instant sticky via CSS `sticky`. |
| Page transitions | None in Phase 1. Standard browser navigation. |
| Focus rings | `--ring` color with 2px offset, appears on keyboard focus only (`:focus-visible`) |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Focus management | All interactive elements must have visible focus rings (`:focus-visible`). Focus ring uses `--ring` color at 2px width with 2px offset. |
| Landmark roles | `<header>`, `<main>`, `<footer>`, `<nav>` used in root layout. Trust pages use `<article>`. |
| Skip navigation | "Skip to main content" link as first focusable element, visually hidden until focused. |
| Color contrast | All text meets WCAG AA: `--foreground` on `--background` (ratio > 7:1), `--muted-foreground` on `--background` (ratio > 4.5:1). Verify with OKLCH contrast checker during implementation. |
| Aria labels | Mobile hamburger: `aria-label="Open navigation menu"`. Disclaimer: `role="note"` with `aria-label`. Breadcrumb: `<nav aria-label="Breadcrumb">`. |
| Image alt text | All images (if any added) must have descriptive alt text. Decorative images use `alt=""`. |
| Heading hierarchy | One `<h1>` per page. Headings never skip levels. |
| Reduced motion | Respect `prefers-reduced-motion: reduce`. Disable Sheet animations and any future Motion animations when active. |
