> Visual and interaction contract for the Calculator Engine & Core Tools phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (multiples of 4, 8-point grid) -- inherited from Phase 1:

| Token | Value | Usage in Phase 2 |
|-------|-------|-------------------|
| xs | 4px | Slider track height, icon-to-label gaps, badge internal padding, hint text margin-top |
| sm | 8px | Input-slider vertical gap, input field groups, chart legend item gaps |
| md | 16px | Input section internal padding, result card internal padding, chart-to-summary gap |
| lg | 24px | Input panel section gaps (between input groups), result card-to-card gaps, action callout card padding |
| xl | 32px | Gap between input panel and results panel (desktop two-column), section breaks within results |
| 2xl | 48px | Calculator page top/bottom padding, gap between chart area and interpretation section |
| 3xl | 64px | Not used in Phase 2 calculator pages (reserved for full-page-level spacing) |

Exceptions:
- Slider thumb touch target: 44x44px tap area (WCAG 2.5.8) with 20px visual thumb diameter. The 44px tap area is achieved via transparent padding around the thumb, not the visible element.
- Number input field width: 112px (28 x 4) fixed width for right-aligned currency/percent fields.

**Source:** Phase 1 UI-SPEC spacing scale. Exceptions derived from WCAG touch target requirements and CONTEXT.md D-05 (slider + input combo).

## Typography

Inherited from Phase 1 with Phase 2-specific role additions:

| Role | Size | Weight | Line Height | Phase 2 Usage |
|------|------|--------|-------------|---------------|
| Body | 16px | 400 (regular) | 1.6 | Interpretation text ("friendly mentor" summary), input labels, hint text body |
| Label | 14px | 400 (regular) | 1.4 | Input hint text, chart axis labels, "US average" contextual labels, comparison delta labels, expandable section triggers |
| Heading | 24px | 600 (semibold) | 1.3 | Calculator page title (H1), section headings ("Results", "Amortization Schedule"), comparison section heading |
| Display | 36px | 600 (semibold) | 1.15 | Not used on individual calculator pages. Used on calculator index page heading only. |

**Phase 2 additions (within existing scale):**

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Result number | 28px | 600 (semibold) | 1.2 | Primary result metric in result cards (e.g., "$2,528/mo"). Desktop only -- scales to 24px on mobile. |
| Result label | 14px | 400 (regular) | 1.4 | Label above result numbers in result cards (e.g., "Monthly Payment") |
| Hint | 12px | 400 (regular) | 1.4 | Input range hints (e.g., "Typical range: $100K--$1M"), "Unusual value" warnings, chart tooltip text |

**Note:** Result number at 28px and hint at 12px are additions to the Phase 1 type scale. Both use existing weights (400, 600). The 28px result number uses Tailwind `text-[28px]` or the closest utility (`text-2xl` maps to 24px, so use explicit 28px). The 12px hint uses `text-xs`.

**Source:** Phase 1 UI-SPEC typography. Result number size chosen for visual prominence as the primary output. 12px hint size for non-essential contextual information that must not compete with input labels.

## Color

All colors use the existing OKLCH CSS custom properties from Phase 1 `globals.css`. No new color definitions needed.

### Light Mode

| Role | CSS Variable | OKLCH Value | Usage in Phase 2 |
|------|-------------|-------------|-------------------|
| Dominant (60%) | `--background` | `oklch(0.99 0.002 240)` | Calculator page background, input panel background, results area background |
| Secondary (30%) | `--secondary` | `oklch(0.93 0.01 240)` | Calculator index page cards, advanced inputs collapsed section background, result card backgrounds, comparison table header row |

| Accent | `--accent` | `oklch(0.55 0.12 175)` | See reserved list below |
| Muted | `--muted` | `oklch(0.95 0.005 240)` | Input panel background on desktop (subtle differentiation from results), expandable detail table row striping (alternating) |

| Foreground | `--foreground` | `oklch(0.15 0.02 240)` | Input labels, result numbers, interpretation body text, table cell text |
| Destructive | `--destructive` | `oklch(0.55 0.2 25)` | Comparison delta badges showing higher costs (red), validation extreme-value warning border |
| Border | `--border` | `oklch(0.88 0.01 240)` | Input field borders, result card borders, comparison table borders, slider track unfilled portion |

### Chart Colors (5-series palette from globals.css)

| Variable | OKLCH Value | Calculator Usage |
|----------|-------------|------------------|
| `--chart-1` | `oklch(0.35 0.1 245)` | Primary data series (principal, savings, Scenario A). Blue. |
| `--chart-2` | `oklch(0.55 0.12 175)` | Secondary data series (interest, expenses, Scenario B). Teal. |
| `--chart-3` | `oklch(0.65 0.1 140)` | Tertiary series (taxes, insurance, third comparison). Green. |
| `--chart-4` | `oklch(0.45 0.08 260)` | Quaternary series (PMI, additional breakdowns). Indigo. |
| `--chart-5` | `oklch(0.75 0.08 200)` | Quinary series (if needed for 5-way breakdowns). Light blue. |

### Accent Reserved For (Phase 2 -- extends Phase 1 list)

Phase 1 accent elements remain. Phase 2 adds:

1. Comparison delta badges showing savings/lower costs (green/teal positive deltas)
2. "Compare" toggle button active state (when comparison mode is ON)
3. Action callout card icons (linking to guides)
4. Calculator index page "Try it" links on each calculator card
5. Chart tooltip accent highlight on hover

Accent is NOT used for: slider tracks (use `--primary`), input field focus rings (use `--ring`), result card borders (use `--border`), table headers (use `--secondary`).

### Semantic Colors for Comparison Deltas

| Meaning | Color | CSS Variable | Usage |
|---------|-------|-------------|-------|
| Positive delta (savings, lower cost) | Teal | `--accent` | Badge background at 10% opacity, badge text at full color |
| Negative delta (higher cost) | Red | `--destructive` | Badge background at 10% opacity, badge text at full color |
| Neutral delta (no change) | Gray | `--muted-foreground` | Badge with muted styling |

**Implementation:** Use Badge component with custom className: `bg-accent/10 text-accent` for positive, `bg-destructive/10 text-destructive` for negative, `bg-muted text-muted-foreground` for neutral.

**Source:** Phase 1 globals.css (color definitions), CONTEXT.md D-10 (green for savings, red for higher costs), D-16 (brand blue + teal accent for charts).

### Interpretation Copy Template

The "friendly mentor" interpretation follows this pattern for all calculators:

> With [input summary], you would [primary result in plain language] -- that is about [contextual comparison to US median/average]. [One sentence of actionable insight].

**Examples:**
- Mortgage: "With a $415K home at 6.5%, you would pay $2,528 per month -- that is about 36% of the median US household income. Consider a 15-year term to save $186K in total interest."
- Budget: "Based on $84K annual income, the 50/30/20 rule suggests $3,500 for needs, $2,100 for wants, and $1,400 for savings each month."

## Interaction Contract

### Slider + Input Combo (D-05)

| Interaction | Behavior |
|-------------|----------|
| Drag slider thumb | Number field updates in real time. Results recalculate after 200ms throttle. URL updates with `history: "replace"`. |
| Type in number field | Slider position updates on blur or Enter key. Value clamped to min/max on blur. Results recalculate immediately. |
| Type extreme value in number field | Border changes to `border-amber-400` (soft amber). Hint text appends "Unusual value" in `text-amber-600`. User is NOT blocked from proceeding. |
| Tab through inputs | Focus moves from number field to next label/field. Slider is focusable via keyboard (arrow keys adjust value by `step`). |
| Touch slider on mobile | 44px touch target. Smooth drag. Haptic feedback if device supports it (no custom implementation -- browser default). |

### Real-Time Calculation (CALC-02)

| Interaction | Behavior |
|-------------|----------|
| Any input value change | Results recalculate within 200ms. No loading spinner for calculations (instant). Chart re-renders with smooth 300ms transition (via Recharts animation). |
| First page load with URL params | Calculator loads with URL param values. If no params, loads with national average defaults. Results display immediately. |
| First page load without URL params | All inputs show default values. "US average" hint labels visible. Results calculated from defaults. |

### Compare Mode (D-09 through D-12)

| Interaction | Behavior |
|-------------|----------|
| Toggle "Compare scenarios" ON (desktop) | Scenario B input column slides in from right (200ms ease-out). Input panel width does not change -- B inputs appear below A inputs in the same panel. Results area splits to show comparison table. |
| Toggle "Compare scenarios" ON (mobile) | Tabs component appears with "Your Scenario" and "Alternative Scenario" tabs. Delta summary card appears at top of results. |
| Toggle "Compare scenarios" OFF | Scenario B inputs and comparison results removed. URL `compare` param set to false. Scenario B params removed from URL. |
| Change Scenario B input | Comparison table and delta badges update in real time (same 200ms throttle as Scenario A). |

### Chart Interactions (D-16)

| Interaction | Behavior |
|-------------|----------|
| Hover chart data point | Tooltip appears with formatted value (currency or percentage). Tooltip background: `--card` with `--border` border. Tooltip text: `--foreground`. |
| Touch chart data point (mobile) | Same tooltip appears on tap. Tap elsewhere to dismiss. |
| Chart animation on data change | Smooth 300ms transition for data updates. Use `animationDuration={300}` on Recharts components. |
| Chart resize (viewport change) | ResponsiveContainer handles automatically. No custom resize logic. |

### Expandable Detail Table (D-15)

| Interaction | Behavior |
|-------------|----------|
| Click "Show full breakdown" | Table expands with 200ms slide-down animation. Trigger text changes to "Hide full breakdown". |
| Click "Hide full breakdown" | Table collapses with 200ms slide-up animation. Trigger text reverts. |
| Table column sort (if applicable) | Click column header to toggle ascending/descending sort. Sort indicator arrow in header. |

### CSV Export (D-10)

| Interaction | Behavior |
|-------------|----------|
| Click "Export to CSV" | Browser download dialog immediately. File named `[calculator-slug]-comparison.csv`. UTF-8 with BOM for Excel compatibility. |
| Export with no comparison data | Button disabled (ghost variant, reduced opacity). Tooltip: "Enable comparison mode to export data." |

### Share Results

| Interaction | Behavior |
|-------------|----------|
| Click "Share Results" | Copies current page URL (with all nuqs params) to clipboard. Button text changes to "Link copied!" for 2 seconds, then reverts. No modal or share sheet. |

### Navigation

| Interaction | Behavior |
|-------------|----------|
| Calculator index card click | Navigates to `/calculators/[slug]`. Full page navigation. |
| Action callout card click | Navigates to guide page (placeholder in Phase 2). Standard link behavior. |
| Browser back after slider change | Returns to previous URL state. Calculator renders with previous param values (nuqs handles this automatically via URL history). |

### Animations Summary

| Element | Animation | Duration | Easing |
|---------|-----------|----------|--------|
| Chart data update | Recharts built-in | 300ms | ease-in-out |
| Comparison panel toggle | CSS height transition | 200ms | ease-out |
| Collapsible advanced section | CSS height transition (via Collapsible component) | 200ms | ease-out |
| Expandable detail table | CSS height transition | 200ms | ease-out |
| Tooltip appear | CSS opacity + transform | 150ms | ease-out |
| "Link copied!" feedback | Text swap (no animation) | 2000ms display | -- |
| Delta badge appear | CSS fade-in | 150ms | ease-out |

**Reduced motion:** All animations respect `prefers-reduced-motion: reduce`. When active, transitions are instant (0ms duration). Recharts `isAnimationActive={false}` when reduced motion is preferred.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Slider ARIA | `aria-label="[Input Label]"`, `aria-valuemin`, `aria-valuemax`, `aria-valuenow`, `aria-valuetext` (formatted value, e.g., "$415,000"). Keyboard: arrow keys adjust by step, Page Up/Down adjust by 10x step. |
| Number input | `aria-label="[Input Label] value"`, `inputmode="decimal"` for percent fields, `inputmode="numeric"` for currency/years. Associated with slider via shared label. |
| Chart | `role="img"` on chart container, `aria-label="[Chart title]: [brief data summary]"`. Hidden from screen readers via `aria-hidden="true"` on SVG internals. Accessible data table alternative provided in the expandable detail table. |
| Comparison table | Standard `<table>` with `<thead>`, `<th scope="col">`, `<th scope="row">`. Delta badges include screen reader text: "saves $186,000" or "costs $43 more per month". |
| Compare toggle | `role="switch"`, `aria-checked`, `aria-label="Compare two scenarios"` |
| Collapsible | `aria-expanded` on trigger, `aria-controls` pointing to content ID |
| Color contrast | Delta badge text (accent on accent/10 background, destructive on destructive/10 background) meets WCAG AA 4.5:1. Verify during implementation. |
| Focus management | Focus ring uses `--ring` color. Tab order: inputs top-to-bottom, then results area. Compare toggle is reachable via tab. |
| Landmark roles | Calculator page: `<main>` wraps all content. Input panel and results panel are `<section>` elements with `aria-label`. |
