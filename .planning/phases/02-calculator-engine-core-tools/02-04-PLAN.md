**1. mortgage.ts** (TOOL-01):

Export `calculateMortgage(params: Record<string, number>): CalculatorResults`

Input params (from nuqs urlKeys): `price`, `dp` (down payment %), `rate`, `term`, `tax` (property tax %), `ins` (annual insurance)

Computation:
- loanAmount = price * (1 - dp/100) using Decimal
- Monthly P&I using PMT formula: M = P * [r(1+r)^n] / [(1+r)^n - 1] where r = rate/100/12, n = term*12
- Handle zero rate edge case: M = P / n
- Monthly property tax = price * (tax/100) / 12
- Monthly insurance = ins / 12
- Total monthly payment = P&I + property tax + insurance
- Total interest = (monthly P&I * term * 12) - loanAmount
- Total cost = (monthly payment * term * 12) + downPayment

Chart data -- amortization (dataKey: "amortization"):
- Generate year-by-year data (not month-by-month for chart -- too many points)
- For each year: compute cumulative principal paid, cumulative interest paid, remaining balance
- Format: `[{ year: 1, principal: X, interest: Y, balance: Z }, ...]`

Chart data -- paymentBreakdown (dataKey: "paymentBreakdown"):
- Pie data: `[{ name: 'Principal & Interest', value: monthlyPI }, { name: 'Property Tax', value: monthlyTax }, { name: 'Insurance', value: monthlyIns }]`

Detail rows: month-by-month amortization schedule (360 rows for 30yr) with columns: Month, Payment, Principal, Interest, Balance

Interpretation template:
`With a ${formatCurrency(price)} home at ${rate}%, you would pay ${formatCurrency(monthlyPayment)} per month -- that is about ${incomePercent}% of the median US household income. ${savingsHint}`
where incomePercent = (monthlyPayment / 7000 * 100).toFixed(0), and savingsHint suggests shorter term if applicable

Outputs: monthlyPayment, totalInterest, totalCost, loanAmount

**3. compound-interest.ts** (TOOL-03):

Export `calculateCompoundInterest(params: Record<string, number>): CalculatorResults`

Input params: `initial`, `monthly`, `rate`, `years`

Computation using decimal.js-light:
- For each year (1 to years): compute balance using FV formula with monthly compounding
  - yearBalance = initial * (1 + rate/100/12)^(year*12) + monthly * [((1+rate/100/12)^(year*12) - 1) / (rate/100/12)]
- Handle zero rate: yearBalance = initial + monthly * year * 12
- futureValue = final year balance
- totalContributions = initial + monthly * years * 12
- totalInterestEarned = futureValue - totalContributions

Chart data -- growth (dataKey: "growth"):
- Area chart data: `[{ year: 0, contributions: initial, interest: 0, balance: initial }, { year: 1, contributions: initial + monthly*12, interest: ..., balance: ... }, ...]`

Outputs: futureValue, totalContributions, totalInterestEarned

Interpretation: `Starting with ${formatCurrency(initial)} and adding ${formatCurrency(monthly)} monthly at ${rate}% return, your investment would grow to ${formatCurrency(futureValue)} in ${years} years -- that is ${formatCurrency(totalInterestEarned)} in earnings on ${formatCurrency(totalContributions)} contributed.`

**1. loan.ts** (TOOL-04):

Export `calculateLoanRepayment(params: Record<string, number>): CalculatorResults`

Input params: `amount`, `rate`, `term` (years), `extra` (extra monthly payment)

Computation using decimal.js-light:
- Standard monthly payment via PMT formula (same as mortgage P&I)
- Without extra payments: straightforward amortization over term*12 months
- With extra payments: generate amortization month by month, applying extra to principal each month
  - actualMonthlyPayment = standardPayment + extra
  - Each month: interest = balance * monthlyRate, principal = min(actualMonthlyPayment - interest, balance), balance -= principal
  - Stop when balance <= 0
  - payoffMonths = number of months until balance reaches 0
  - interestSaved = (standardTotalInterest) - (actualTotalInterest)

Chart data -- balanceOverTime (dataKey: "balanceOverTime"):
- Area chart: `[{ month: 1, standard: balance1, withExtra: balance1_extra }, ...]` showing both curves overlaid

Outputs: monthlyPayment, totalInterest, totalCost, payoffMonths, interestSaved (0 if no extra)

Detail rows: month-by-month schedule with columns: Month, Payment, Principal, Interest, Extra, Balance

Interpretation: `A ${formatCurrency(amount)} loan at ${rate}% over ${term} years costs ${formatCurrency(monthlyPayment)}/mo.${extra > 0 ? ` Adding ${formatCurrency(extra)}/mo extra saves ${formatCurrency(interestSaved)} in interest and pays off ${monthsSaved} months early.` : ' Consider extra payments to save on interest.'}`

**2. savings.ts** (TOOL-05):

Export `calculateSavingsGoal(params: Record<string, number>): CalculatorResults`

Input params: `goal`, `current`, `rate`, `years`

Computation using decimal.js-light:
- Required monthly contribution to reach goal:
  - If rate is 0: monthlyRequired = (goal - current) / (years * 12)
  - Else: Use inverse of FV formula. FV = PV*(1+r)^n + PMT*((1+r)^n - 1)/r, solve for PMT
  - PMT = (FV - PV*(1+r)^n) / (((1+r)^n - 1)/r)
  - where r = rate/100/12, n = years * 12, FV = goal, PV = current
- Generate year-by-year projection: for each year, compute projected balance with monthlyRequired contributions
- projectedTotal at end (should be close to goal)
- totalContributed = current + monthlyRequired * years * 12
- interestEarned = projectedTotal - totalContributed

Chart data -- savingsProgress (dataKey: "savingsProgress"):
- Line chart: `[{ year: 0, balance: current, goal: goal }, { year: 1, balance: ..., goal: goal }, ...]`
- ReferenceLine at goal amount

Outputs: monthlyRequired, projectedTotal, totalContributed, interestEarned

Interpretation: `To reach ${formatCurrency(goal)} in ${years} years starting with ${formatCurrency(current)}, save ${formatCurrency(monthlyRequired)} per month at ${rate}% return. You will contribute ${formatCurrency(totalContributed)} and earn ${formatCurrency(interestEarned)} in interest.`
