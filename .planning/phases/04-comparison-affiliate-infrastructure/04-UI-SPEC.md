## Spacing Scale

Declared values (multiples of 4, 8-point grid) -- inherited from Phases 1-3:

|-------|-------|-------------------|
| xs | 4px | Icon-to-label gaps in badge components, table cell inline padding between icon and text, filter dropdown icon gap |

- Methodology page content max-width: 720px (same as trust pages and guide article column), optimized for long-form prose readability.

**Source:** Phase 1-3 UI-SPEC spacing scales. Layout widths match existing patterns (1200px for multi-column, 720px for prose).

## Typography

4 declared sizes, 2 weights. Inherited from Phase 3 base scale with Phase 4 role assignments.

| Role | Size | Weight | Line Height | Phase 4 Usage |
|------|------|--------|-------------|---------------|
| Label | 14px | 400 (regular) | 1.4 | Table header column labels, filter dropdown labels, product card attribute labels (mobile), "Best For" badge text, affiliate disclosure body text, methodology page "Last updated" date, table cell secondary text (e.g., fee ranges, term ranges) |
| Body | 16px | 400 (regular) | 1.6 | Comparison page intro paragraph, product card descriptions, methodology page body text, comparison index page category descriptions, product detail attribute values in table cells |
| Heading | 24px | 600 (semibold) | 1.3 | Comparison category page H1 ("Credit Card Comparison"), comparison index page H1 ("Product Comparisons"), methodology page H1 ("How We Rank Products"), methodology page H2 section headings |
| Subheading | 20px | 600 (semibold) | 1.3 | Methodology page H3 headings (per-category criteria sections), product category card titles on comparison index page |

**Body line height at 1.6:** Comparison pages have shorter prose passages (intro paragraphs, methodology text) compared to Phase 3 guides (1.7). The 1.6 ratio matches Phase 1-2 content pages and is appropriate for these shorter blocks.

**Table text:** All table cell text uses `text-sm` (14px). Table headers use `text-sm font-medium` (14px, 500 weight -- inherited from shadcn Table component defaults). The font-medium weight on table headers is a shadcn default, not a new weight declaration -- it is baked into the `TableHead` component.

**Weights:** 400 (regular) and 600 (semibold). No other weights used in custom code.

**Responsive scaling:**
- Heading: 24px on desktop, 20px on mobile (< md)
- Subheading: 20px on desktop, 18px on mobile (< md)
- Body and Label: No responsive changes

**Implementation:** 14px = `text-sm`. 16px = `text-base`. 20px = `text-xl`. 24px = `text-2xl`. 18px = `text-lg`.

**Source:** Phase 3 UI-SPEC typography. Body line height adjusted from 1.7 (guides) to 1.6 (shorter content blocks). Table typography inherited from shadcn Table component.

## Interaction Contract

### Sortable Column Headers (D-04)

| Interaction | Behavior |
|-------------|----------|
| Click unsorted column header | Sorts table by that column ascending. Sort icon changes from `ArrowUpDown` to `ArrowUp`. |
| Click ascending-sorted column header | Toggles to descending. Sort icon changes to `ArrowDown`. |
| Click descending-sorted column header | Returns to unsorted (default order). Sort icon changes back to `ArrowUpDown`. |
| Keyboard | Column headers are `<button>` elements. Enter/Space triggers sort. Tab navigates between headers. |
| Screen reader | `aria-sort="ascending"`, `aria-sort="descending"`, or `aria-sort="none"` on `<th>`. Sort button has `aria-label="Sort by {column name}"`. |
| URL state | Active sort column and direction stored in URL via nuqs (`?sort=apr&dir=asc`). Shareable and bookmarkable. |

### Category-Specific Filters (D-05)

| Interaction | Behavior |
|-------------|----------|
| Open filter dropdown | Select component opens. Shows available filter values. |
| Select filter value | Table updates immediately (no submit button). URL params update via nuqs. Product count updates. |
| Multiple filters applied | Filters combine with AND logic. Each filter narrows the result set. |
| Clear individual filter | Select the "All" / default option in the dropdown. |
| Clear all filters | "Clear filters" button resets all filter params. Only visible when at least one filter is active. |
| Mobile filter layout | Filters stack vertically. Sort dropdown appears first, then category filters. |
| URL state | All filter values stored in URL via nuqs (`?rewards=travel&score=750`). Shareable and bookmarkable. |
| Zero results after filtering | Table/cards replaced with empty state: "No matching products" message with "clear all filters" link. |

### Animations Summary

| Element | Animation | Duration | Easing |
|---------|-----------|----------|--------|
| Table row hover | CSS background-color transition (shadcn TableRow default) | 150ms | ease-out |
| Sort icon change | CSS opacity transition | 100ms | ease-out |
| Card hover border color | CSS border-color transition | 150ms | ease-out |

**No Motion (framer-motion) animations in Phase 4.** Table reordering, filter updates, and card reordering use instant DOM updates. Animated reordering of product rows would hurt perceived performance on large datasets and add unnecessary client JS.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
