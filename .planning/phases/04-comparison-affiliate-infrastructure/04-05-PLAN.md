Props: `{ products: Record<string, unknown>[]; category: ProductCategory; lastVerified: string; }`

Inside the client component:
- Use `useQueryStates(categoryParams[category])` to get sort/filter state from URL
- Extract sort column and direction from state (the `sort` and `dir` keys)

For insurance columns: Name, Monthly Premium, Coverage Level, Deductible Range, Coverage Highlights, CTA

- Define attribute configs per category for mobile cards (same data, different format)
- Render sort toggle handler that cycles: asc -> desc -> null (per UI-SPEC: click unsorted -> asc, click asc -> desc, click desc -> unsorted)
- Handle filter change: update URL state via nuqs setState
- Handle clear all: reset all filter params to 'all'
- Show empty state when filteredProducts.length === 0: "No matching products" with clear filters link (per UI-SPEC Empty States)
