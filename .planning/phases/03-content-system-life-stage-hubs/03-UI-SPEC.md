# Phase 3 -- UI Design Contract

> Visual and interaction contract for the Content System & Life-Stage Hubs phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui CLI v4 |
| Preset | base-nova (Base UI primitives, Tailwind v4, OKLCH, neutral base) |
| Component library | Base UI (via shadcn/ui base-nova style) |
| Icon library | Lucide React |
| Font | Inter (self-hosted via `next/font/google`, variable `--font-inter`) |

**Existing components (from Phases 1-2):** button, navigation-menu, sheet, breadcrumb, card, badge, separator, accordion, alert, tooltip, slider, input, label, tabs, switch, collapsible, table, select.

**New components for Phase 3:** `npx shadcn@latest add popover`

**Why Popover instead of Tooltip for glossary terms:** The existing tooltip component uses a dark background (`bg-foreground`) designed for single-line hints. Glossary definitions are 2-3 sentences long and need a light background, more content space, and mobile-friendly tap activation. Popover provides all of this.

**Source:** components.json (`style: "base-nova"`), Phase 2 UI-SPEC, RESEARCH.md Pitfall 4.

## Spacing Scale

Declared values (multiples of 4, 8-point grid) -- inherited from Phases 1-2:

| Token | Value | Usage in Phase 3 |
|-------|-------|-------------------|
| xs | 4px | Icon-to-label gaps in callout boxes, glossary term popover internal padding, TOC item indent, badge internal padding |
| sm | 8px | TOC link vertical spacing, callout box icon-to-text gap, hub tip list item gaps, guide FAQ answer spacing |
| md | 16px | Callout box internal padding (horizontal), guide body paragraph spacing, hub section internal padding, guide CTA banner padding |
| lg | 24px | Callout box internal padding (vertical), TOC sidebar internal padding, hub card grid gaps, guide-to-guide index card gaps |

**Source:** Phase 1/2 UI-SPEC spacing scales. Phase 3 exceptions from RESEARCH.md Pattern 6, CONTEXT.md D-02.

## Typography

4 declared sizes, 2 weights. Inherited from Phase 2 base scale with Phase 3 role assignments.

| Role | Size | Weight | Line Height | Phase 3 Usage |
|------|------|--------|-------------|---------------|
| Label | 14px | 400 (regular) | 1.4 | TOC links, reading time badge text, "Last updated" date, glossary page term slug text, hub section subheading intro text, callout box label ("Key Takeaway", "Pro Tip"), guide card meta (reading time, category) |
| Body | 16px | 400 (regular) | 1.7 | Guide MDX body text, hub intro paragraphs, glossary term definitions (popover and standalone page), hub tip descriptions, callout box body text, FAQ answer text |
| Heading | 24px | 600 (semibold) | 1.3 | Guide page H1, hub page H1, glossary page H1, guide index page H1, hub index page H1, hub section headings ("Your Calculators", "Guides For You"), guide H2 headings |
| Subheading | 20px | 600 (semibold) | 1.3 | Guide H3 headings, hub tip section heading ("Quick Tips"), hub "Next Steps" heading, glossary letter group headings (A, B, C...) |

**Guide body line height at 1.7:** Long-form educational content at 2000-3000 words requires generous line spacing for sustained reading. The 1.7 ratio follows established typographic best practices for long-form article readability, distinct from the 1.6 used for shorter paragraph text in Phases 1-2.

**Weights:** 400 (regular) and 600 (semibold). No other weights used.

**Responsive scaling:**
- Heading: 24px on desktop, 20px on mobile (< md)
- Subheading: 20px on desktop, 18px on mobile (< md)
- Body and Label: No responsive changes

**Implementation:** 14px = `text-sm`. 16px = `text-base`. 20px = `text-xl`. 24px = `text-2xl`. Guide body uses `text-base leading-[1.7]`. H2 uses `mt-10 mb-4 text-xl font-semibold` (desktop) per RESEARCH.md mdx-components pattern. H3 uses `mt-8 mb-3 text-lg font-semibold`.

**Source:** Phase 2 UI-SPEC typography. Subheading role replaces Phase 2 "Result" role (28px) which is not applicable in Phase 3. Guide body line height from RESEARCH.md Pattern 4 code example.

### Accent Reserved For (Phase 3 -- extends Phase 2 list)

Phase 1 and 2 accent elements remain. Phase 3 adds:

1. Glossary term dotted underline on hover state (`border-muted-foreground/50` at rest, `border-accent` on hover)
2. Guide CTA banner "Try the calculator" link/button
3. Guide index page card "Read guide" links
4. Hub index page card "Explore hub" links
5. Active TOC link left-border indicator
6. Inline content links within guide MDX body (`text-accent underline`)
7. MDX blockquote left border (`border-accent/30`)

Accent is NOT used for: callout box backgrounds (use primary/chart-3/accent at 5-10% opacity), hub section headings (use `--foreground`), TOC heading text (use `--foreground`), glossary term text at rest (use inherited color with dotted underline).

## Interaction Contract

### Table of Contents (D-02)

| Interaction | Behavior |
|-------------|----------|
| Page load (desktop) | TOC extracted from `article h2[id], h3[id]` elements via DOM query. Rendered as anchor links in sticky sidebar. |
| Scroll through article | IntersectionObserver tracks which heading is in viewport. Active heading link gets `text-foreground font-semibold` + left border indicator in `--accent`. Inactive links use `text-muted-foreground`. |
| Click TOC link | Smooth scroll to heading. URL hash updated. Active state updates immediately. |
| Mobile TOC | Collapsed by default. Tap "On this page" to expand horizontal list of H2 anchor links. Tap a link to scroll and close. |
| Resize from mobile to desktop | TOC transitions from collapsed horizontal to sticky sidebar. No flash or layout shift. |

### Glossary Term Popover (D-10, D-12)

| Interaction | Behavior |
|-------------|----------|
| Hover glossary term (desktop) | Popover appears after 200ms delay. Shows term name (semibold) + definition (regular). Max width 280px. |
| Click glossary term (desktop) | Popover toggles open/closed. Stays open until clicked again or clicked outside. |
| Tap glossary term (mobile) | Popover opens on first tap. Closes on tap outside. No hover state on touch devices. |
| Term not found in glossary | Term renders as plain `<span>` without dotted underline or popover. Silent fallback -- no error displayed. |
| Multiple terms open | Only one popover open at a time. Opening a new term closes the previous one (default Popover behavior). |

### Guide CTA Banner (D-04)

| Interaction | Behavior |
|-------------|----------|
| Desktop (sidebar) | CTA card positioned at top of TOC sidebar. `sticky top-24`. Always visible while reading. |
| Click "Open [calculator]" button | Navigates to `/calculators/[slug]`. Standard link navigation. |
| Mobile | CTA banner appears as a full-width card between guide header and article content. Not sticky on mobile (saves viewport space). |

### Hub Navigation

| Interaction | Behavior |
|-------------|----------|
| Hub index card click | Navigates to `/hubs/[slug]`. Full page navigation. |
| Hub calculator card click | Navigates to `/calculators/[slug]`. |
| Hub guide card click | Navigates to `/guides/[slug]`. |
| Hub "Next Steps" card click | Navigates to `/hubs/[slug]`. |
| Hub icon rendering | Uses `getIcon()` pattern from existing `action-callout.tsx` -- dynamic Lucide icon import from icon name string. |

### Guide Index Navigation

| Interaction | Behavior |
|-------------|----------|
| Guide card click | Navigates to `/guides/[slug]`. Full page navigation. |
| Guide card hover (desktop) | Card border transitions to `border-accent/50` over 150ms (same pattern as calculator action callout cards). |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Guide article structure | `<article>` wraps all MDX content. One `<h1>` per page. H2/H3 headings never skip levels. Headings have `id` attributes (via rehype-slug). |
| TOC navigation | `<nav aria-label="Table of contents">`. TOC links use `<a href="#heading-id">`. Active link has `aria-current="true"`. |
| Glossary term popover | `<button>` trigger (not `<span>`) for keyboard accessibility. Popover has `role="dialog"`. Trigger has `aria-expanded` and `aria-haspopup="dialog"`. Escape key closes popover. |
| Callout boxes | `role="note"` on callout container. `aria-label="Key Takeaway"` (or "Pro Tip" / "Definition"). |
| FAQ accordion | `aria-expanded` on question trigger. `aria-controls` pointing to answer ID. Questions are `<button>` elements. Keyboard: Enter/Space toggles. |
| Hub icon | Decorative -- `aria-hidden="true"` on Lucide icon SVG. Hub name provides the meaningful text. |
| Reading time | Wrapped in `<span>` -- no special ARIA needed. Part of visible meta row. |
| Color contrast | Callout box text on tinted backgrounds meets WCAG AA 4.5:1. Hub accent colors on white backgrounds meet WCAG AA. Glossary term dotted underline is supplementary to the cursor change (not relied upon solely). |
| Focus management | All interactive elements (TOC links, popover triggers, accordion triggers, card links) have visible focus rings using `--ring`. Tab order follows DOM order. |
| Landmark roles | Guide page: `<article>` for content, `<nav>` for TOC, `<aside>` for CTA banner. Hub page: `<main>` wraps all content, each section is `<section>` with `aria-label`. |
| Heading hierarchy | Guides: H1 (title) > H2 (sections) > H3 (subsections). Hubs: H1 (hub name) > H2 (section headings). Glossary: H1 (page title) > H2 (letter groups). |
| Reduced motion | Smooth scroll, popover transitions, accordion animations all respect `prefers-reduced-motion`. |
